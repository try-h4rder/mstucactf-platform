## Web

Современный веб основан на **клиент-серверной архитектуре**, где клиент (обычно браузер) отправляет запросы к серверу, а сервер отвечает, передавая нужные данные. Основой взаимодействия между ними является протокол **HTTP (HyperText Transfer Protocol)** — прикладной протокол передачи гипертекста, то есть веб-страниц, изображений, скриптов и других ресурсов. HTTP определяет структуру запросов и ответов, а также правила обмена данными между браузером и сервером.

Каждый HTTP-запрос содержит метод (например, **GET** для получения данных или **POST** для их отправки), путь к ресурсу, версию протокола и набор заголовков. Ответ сервера состоит из кода состояния (например, 200 — успех, 404 — страница не найдена), заголовков и тела с содержимым страницы. На основе этого обмена формируется работа всего веб-пространства — от простых сайтов до сложных веб-приложений.

Язык **HTML (HyperText Markup Language)** используется для разметки веб-страниц, позволяя браузеру интерпретировать и отображать информацию в виде текста, изображений, ссылок и интерактивных элементов. Веб-технологии объединяют принципы сетевого взаимодействия, структурирования данных и безопасности, формируя фундамент современного Интернета.

<quiz>
Ваш личный чат с другом в веб‑версии мессенджера открывается по ссылке:
```
https://web.telegram.org/a/#13371337
```
Что из перечисленного ниже является веб‑уязвимостью?

- [x] Изменение числа `13371337` открывает личные чаты чужих пользователей
- [ ] Попытка сменить протокол на `http` приводит к ошибке мессенджера
- [ ] Ссылка на личный чат с другом для обоих собеседников общая
- [ ] Ссылка на личный чат с другом для обоих собеседников индивидуальная

IDOR (Insecure Direct Object Reference) — это уязвимость, при которой пользователь может получить доступ к объекту (например, чату или фото) по его идентификатору без проверки прав. Если изменение числа в ссылке позволяет открыть чужой чат, значит сервер не контролирует, кому этот объект действительно доступен.

При этом «общая» или «индивидуальная» ссылка — это не уязвимость, а особенность архитектуры сервиса. Она описывает, как именно разработчики организовали доступ к ресурсам, но сама по себе не говорит о наличии или отсутствии проблемы с безопасностью.
</quiz>

<quiz>
Какие из перечисленных способов отправят HTTP-запрос с заголовком `Best-Anime`?
```shell
# Вариант: curl -H
curl -H "Best-Anime: Attack on Titan" https://ya.ru
```
```shell
# Вариант: curl URL
curl "https://ya.ru?Best-Anime=Attack+on+Titan"
```
```python
# Вариант: python headers
import requests
requests.get('https://ya.ru',headers={'Best-Anime': 'Attack on Titan'})
```
```python
# Вариант: python URL
import requests
requests.get('https://ya.ru?Best-Anime=Attack+on+Titan')
```

- [x] `curl -H`
- [ ] `curl URL`
- [x] `python headers`
- [ ] `python URL`

HTTP-заголовки передаются в специальной секции запроса, редактировать которую можно, например, при помощи ключа `-H` / `--header` в утилите `curl` или словаря `headers` в библиотеке `python-requests`. И да, протокол HTTP допускает использование пользовательских заголовков, например, `Best-Anime`.
</quiz>
